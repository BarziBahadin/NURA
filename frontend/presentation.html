NURA
AI Customer Support

A fully self-hosted, Arabic-first AI support platform — from first message to resolved case, with a human agent always one tap away.

The Problem
Why we built this

Support at scale is expensive and slow

🕐

Long wait times

Customers wait minutes for answers to questions the AI can resolve in seconds.

🔄

Repetitive queries

80% of tickets are the same questions — balance, recharge, SIM activation, packages.

📊

No visibility

No real-time data on what customers are asking, where agents are overwhelmed, or what the AI missed.

The Solution
What NURA is

One platform — three layers

🤖

AI Layer

GPT-powered agent trained on Rcell's knowledge base. Answers in Arabic, Kurdish, and English. Knows when it doesn't know — and escalates.

🧑‍💼

Agent Layer

Real-time handoff queue. Canned replies, session history, customer cases — everything in one admin panel.

📈

Analytics Layer

KPI dashboard, knowledge gap detection, session browser, and full audit trail. Continuously improves the AI.

AI Intelligence
How the AI answers

Two-layer knowledge system

1

Always-loaded Articles

20 core support articles always in the AI's context — packages, billing, SIM, VoLTE.

2

RAG over Handbooks

Vector search over the full call-center handbook. Top 3 relevant chunks added per message.

3

Confidence Threshold

If confidence score is low, the AI flags it as a knowledge gap and escalates gracefully.

4

Human Handoff

Real-time queue handoff to a live agent. Full session history transferred instantly.

Live Queue
When the AI escalates

The agent takes over in seconds

1

AI flags the session

Confidence drops below threshold or customer requests human — session enters the queue instantly.

2

Agent sees it live

Badge count goes up in the sidebar. Queue shows customer's last message, channel, and how long they've been waiting.

3

Claim & reply

Agent claims the session. Full chat history loads instantly. Canned replies are chips — one click to insert.

4

Resolve & log

Agent marks resolved. Session saves to cases. The AI's knowledge gap is flagged for admin review.

Telegram Integration
Second channel — same AI brain

Full support on Telegram

📋
Inline topic tree
Customers navigate support topics via Telegram keyboard buttons — the same topic tree as the web widget. No free-text typing required.
🤖
Same AI, same knowledge base
Every message goes through the same FastAPI → GPT → RAG pipeline. Answers are identical across web and Telegram — no separate AI to maintain.
🧑‍💼
Handoff to the same admin queue
Escalation from Telegram pushes the session into the exact same live queue. The agent replies from the web panel — the customer sees it in their Telegram chat.
⭐
Vertical star rating keyboard
After resolution the bot sends a 5-row star rating keyboard — one star per row so it's easy to tap on mobile. Ratings flow into the admin analytics dashboard.
Background long-polling worker — telegram_worker.py — no webhooks, no extra infra needed.

Security
Built for production

Security & data privacy

🔐

JWT Authentication

All admin routes protected with signed JWT tokens. Role-based access: admin, agent, viewer.

🏠

Self-Hosted

Your data never leaves your servers. PostgreSQL, Redis, ChromaDB all run on your infrastructure.

🛡️

Rate Limiting

slowapi rate limiting on all public endpoints. DDoS protection built in.

📝

Full Audit Trail

Every session, message, escalation, and case action is logged with timestamps.

Secrets in .env

No hardcoded credentials. All secrets via environment variables, gitignored.

Automated Backups

Daily Postgres dumps + file backups. 7-day retention, scriptable via cron.
